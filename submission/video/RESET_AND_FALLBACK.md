# Reset and Fallback

## Clean-state reset

The sample calendar is session-only, so reloading the page clears it. The
Decision Journal persists in browser storage and must be cleared through the
visible product UI.

1. Open <https://yo4e.github.io/maybe-tomorrow/>.
2. If **Decision Journal** contains entries, select **Clear journal**.
3. Confirm with the second **Clear journal** control.
4. Reload the page.
5. Confirm that the home screen starts with an empty activity field and no
   journal entries.
6. Close unrelated tabs and notifications before recording.

Do not use browser developer tools or edit storage directly. The visible reset
proves the ordinary user path.

## Reset between takes

- A failed core take: select **Try another**, or reload and re-enter the core
  answer key.
- A failed sample take before the verdict: reload, select **Try the sample
  day**, then **View this day**, and repeat the sample answer key.
- A failed sample take after saving: clear the journal through the visible
  controls, reload, and repeat.
- A stale or confusing imported file: reload and use only the bundled sample.

## Preferred fallback if calendar capture fails

The calendar path is optional context, not the product prerequisite. If it
cannot be recorded cleanly:

1. keep the complete home, Quick Check, **Maybe tomorrow.**, and **What would
   need to change?** clips;
2. use the prepared Today Map and room-making screenshots from
   [`../assets/`](../assets/) as brief stills;
3. retain the same narration, making no claim that a live calendar service is
   connected; and
4. spend the recovered time on the repository evidence and closing boundary.

If even the prepared stills cannot be used, omit the detailed room-making
demonstration and say only that optional local calendar context exists. Never
substitute a personal calendar or a fake interaction.

## Safe failure hierarchy

1. Retry with the bundled sample.
2. Use the committed fictional screenshots.
3. Shorten the optional calendar section.
4. Preserve the core verdict, Cost of Yes, GPT-5.6/Codex explanation, and
   closing privacy boundary.
